Adventure Mode offers structured progression through 16 challenging levels with increasing difficulty. The Dynamic Difficulty Adaptation System adjusts parameters based on player performance.

**Implementation:**
```kotlin
// AdventureGameEngine.kt
fun calculateDifficultyAdjustment(
    levelNumber: Int,
    playerStats: PlayerStats
): DifficultyParameters {
    val baseGridSize = getBaseGridSize(levelNumber)
    val baseTimeLimit = getBaseTimeLimit(levelNumber)
    
    // Adjust based on recent performance
    val accuracyModifier = when {
        playerStats.recentAccuracy > 0.90 -> 1.2f  // Increase difficulty
        playerStats.recentAccuracy < 0.60 -> 0.8f  // Decrease difficulty
        else -> 1.0f
    }
    
    return DifficultyParameters(
        gridSize = (baseGridSize * accuracyModifier).toInt(),
        timeLimit = (baseTimeLimit / accuracyModifier).toLong()
    )
}
```

**Key Features:**
- 16 unique challenging levels with progressive scaling
- Detailed performance metrics for each level
- Star rating system (1-3 stars based on performance)
- Level unlocking progression system

#### Multiplayer Mode Features

Multiplayer Mode enables competitive local play for two players using a shared device. Players take turns matching cards on the same grid with real-time score tracking.

**Implementation:**
```kotlin
// MultiplayerGameEngine.kt
fun handleCardSelection(
    gameState: MultiplayerGameState,
    cardIndex: Int
): MultiplayerGameState {
    val updatedState = gameState.copy(
        selectedCards = gameState.selectedCards + cardIndex
    )
    
    if (updatedState.selectedCards.size == 2) {
        val card1 = gameState.cards[updatedState.selectedCards[0]]
        val card2 = gameState.cards[updatedState.selectedCards[1]]
        
        if (card1.id == card2.id) {
            // Match found - award point to current player
            return updatedState.copy(
                matchedCards = updatedState.matchedCards + card1.id,
                scores = updatedState.scores.updateScore(
                    gameState.currentPlayer, 
                    increment = 1
                ),
                selectedCards = emptyList()
            )
        } else {
            // No match - switch players
            return updatedState.copy(
                currentPlayer = gameState.currentPlayer.opponent(),
                selectedCards = emptyList()
            )
        }
    }
    
    return updatedState
}
```

**Key Features:**
- Red vs Blue team identification
- Shared card grid ensures fair play
- Turn-based matching with clear visual indicators
- Final score comparison and winner declaration

### Offline Capabilities

#### Smart Synchronization

Memory Match Madness implements a comprehensive offline-first architecture ensuring uninterrupted gameplay regardless of network connectivity.

**Architecture:**
```
User Action → Local RoomDB (Immediate) → Background Queue → Sync Manager
                                                                    ↓
                                                         Network Available?
                                                                    ↓
                                                    Yes → Firestore (Cloud Sync)
                                                     No → Retry Queue
```

**Implementation Components:**

**1. Room Database - Local Persistence**
```kotlin
// GameResultEntity.kt
@Entity(tableName = "game_results")
data class GameResultEntity(
    @PrimaryKey val id: String = UUID.randomUUID().toString(),
    val userId: String,
    val gameMode: String,
    val score: Int,
    val time: Long,
    val theme: String,
    val syncStatus: SyncStatus = SyncStatus.PENDING,
    val createdAt: Long = System.currentTimeMillis()
)

enum class SyncStatus {
    PENDING, SYNCED, FAILED
}
```

**2. Background Synchronization**
```kotlin
// SyncWorker.kt
class SyncWorker(context: Context, params: WorkerParameters) 
    : CoroutineWorker(context, params) {
    
    override suspend fun doWork(): Result {
        return try {
            val pendingResults = gameResultDao
                .getPendingResults()
            
            pendingResults.forEach { result ->
                syncToFirestore(result)
                gameResultDao.updateSyncStatus(
                    result.id, 
                    SyncStatus.SYNCED
                )
            }
            Result.success()
        } catch (e: Exception) {
            Result.retry()
        }
    }
}
```

**3. Conflict Resolution**
```kotlin
// SyncManager.kt
suspend fun resolveConflict(
    local: GameResultEntity,
    remote: GameResultEntity
): GameResultEntity {
    return when {
        // Cloud data is newer
        remote.createdAt > local.createdAt -> remote
        // Local data is newer
        local.createdAt > remote.createdAt -> local
        // Same timestamp - merge scores
        else -> local.copy(
            score = maxOf(local.score, remote.score),
            syncStatus = SyncStatus.SYNCED
        )
    }
}
```

**Key Features:**
- **Room Database** - Local data persistence for all game modes
- **Background Sync** - Automatic cloud synchronization on network reconnection
- **Conflict Resolution** - Intelligent merging of offline and online data
- **WorkManager** - Scheduled background sync tasks with retry logic
- **Auto-Detection** - Automatic online/offline mode switching with network listener

### Part 2 Innovative Features

#### 1. Dynamic Difficulty Adaptation System

<div align="center">

```
Player Performance → Algorithm Analysis → Auto-Adjustment → Optimal Challenge
```

</div>

The Dynamic Difficulty Adaptation System automatically adjusts game parameters based on player performance to maintain optimal engagement and challenge levels.

**Algorithm Implementation:**
```kotlin
// DifficultyAdapter.kt
class DifficultyAdapter {
    fun calculateNextLevelDifficulty(
        currentLevel: Int,
        performanceHistory: List<LevelPerformance>
    ): DifficultyParameters {
        val recentPerformance = performanceHistory.takeLast(5)
        val avgAccuracy = recentPerformance.map { it.accuracy }.average()
        val avgCompletionTime = recentPerformance.map { it.completionTime }.average()
        
        val baseGridSize = getBaseGridSize(currentLevel)
        val baseTimeLimit = getBaseTimeLimit(currentLevel)
        
        // Adjust based on performance metrics
        val difficultyMultiplier = when {
            avgAccuracy > 0.90 && avgCompletionTime < baseTimeLimit * 0.7 -> 1.25f
            avgAccuracy > 0.80 && avgCompletionTime < baseTimeLimit * 0.85 -> 1.10f
            avgAccuracy < 0.60 || avgCompletionTime > baseTimeLimit -> 0.85f
            else -> 1.0f
        }
        
        return DifficultyParameters(
            gridSize = calculateAdjustedGridSize(baseGridSize, difficultyMultiplier),
            timeLimit = calculateAdjustedTimeLimit(baseTimeLimit, difficultyMultiplier),
            cardRevealTime = adjustCardRevealTime(difficultyMultiplier)
        )
    }
}
```

```kotlin
// StreakManager.kt
class StreakManager(
    private val streakRepository: StreakRepository,
    private val notificationManager: NotificationManager
) {
    suspend fun recordDailyPlay(userId: String) {
        val today = LocalDate.now()
        val streak = streakRepository.getStreak(userId)
        
        if (streak.lastPlayDate == today) {
            return
        }
        
        val updatedStreak = when {
            // Played yesterday - continue streak
            streak.lastPlayDate == today.minusDays(1) -> {
                streak.copy(
                    currentStreak = streak.currentStreak + 1,
                    lastPlayDate = today,
                    longestStreak = maxOf(
                        streak.longestStreak,
                        streak.currentStreak + 1
                    )
                )
            }
            // Grace period active
            streak.lastPlayDate == today.minusDays(2) && 
                streak.gracePeriodsUsed < MAX_GRACE_PERIODS -> {
                streak.copy(
                    currentStreak = streak.currentStreak + 1,
                    lastPlayDate = today,
                    gracePeriodsUsed = streak.gracePeriodsUsed + 1
                )
            }
            // Streak broken
            else -> {
                streak.copy(
                    currentStreak = 1,
                    lastPlayDate = today,
                    gracePeriodsUsed = 0
                )
            }
        }
        
        streakRepository.updateStreak(userId, updatedStreak)
        checkMilestones(userId, updatedStreak)
    }
    
    private suspend fun checkMilestones(userId: String, streak: Streak) {
        val milestones = listOf(3, 5, 10, 30, 50, 100)
        
        if (streak.currentStreak in milestones) {
            val achievement = Achievement.Streak(streak.currentStreak)
            achievementManager.unlockAchievement(userId, achievement)
            
            notificationManager.sendMilestoneNotification(
                userId = userId,
                milestone = streak.currentStreak,
                message = "Incredible! ${streak.currentStreak} day streak!"
            )
        }
    }
}
```

- Milestone achievements at 3, 5, 10, 30, 50, and 100-day streaks

```kotlin
class NotificationScheduler(
    private val workManager: WorkManager,
    private val userPrefsRepository: UserPreferencesRepository
) {
    fun scheduleDailyReminder(userId: String) {
        val prefs = userPrefsRepository.getNotificationPreferences(userId)
        
        if (!prefs.dailyRemindersEnabled) return
        
        val now = LocalTime.now()
        if (isQuietHours(now, prefs)) {
            scheduleAfterQuietHours(userId, prefs)
            return
        }
        
        val delay = calculateDelayUntilReminderTime(
            prefs.preferredReminderTime
        )
        
        val reminderWork = OneTimeWorkRequestBuilder<DailyReminderWorker>()
            .setInitialDelay(delay, TimeUnit.MILLISECONDS)
            .setInputData(workDataOf("user_id" to userId))
            .build()
        
        workManager.enqueueUniqueWork(
            "daily_reminder_$userId",
            ExistingWorkPolicy.REPLACE,
            reminderWork
        )
    }
```

```kotlin
// LanguageManager.kt
class LanguageManager(
    private val context: Context,
    private val userPrefsRepository: UserPreferencesRepository
) {
    enum class SupportedLanguage(val code: String, val displayName: String) {
        ENGLISH("en", "English"),
        AFRIKAANS("af", "Afrikaans"),
        ISIZULU("zu", "IsiZulu")
    }
    
    suspend fun changeLanguage(language: SupportedLanguage) {
        userPrefsRepository.updateLanguagePreference(language.code)
        applyLanguage(language)
        restartActivity()
    }
    
    private fun applyLanguage(language: SupportedLanguage) {
        val locale = Locale(language.code)
        Locale.setDefault(locale)
        
        val config = context.resources.configuration
        config.setLocale(locale)
        
        context.createConfigurationContext(config)
        context.resources.updateConfiguration(
            config,
            context.resources.displayMetrics
        )
    }
}
```

3. Navigate to the cloned directory
4. Click `OK` to open
5. Wait for Gradle sync

#### Step 3: Configure Firebase

1. Visit [Firebase Console](https://console.firebase.google.com)
2. Create or select project
3. Add Android app: `vcmsa.projects.prog7314`
4. Download `google-services.json`
5. Place in `app/` directory
6. Enable Authentication, Firestore, Storage, Cloud Messaging

#### Step 4: Set up API Backend

```bash
cd memory-match-maddness-api
npm install
```

Create `.env` file:
```env
PORT=3000
NODE_ENV=development
FIREBASE_PROJECT_ID=your-project-id
```

Start server:
```bash
npm run dev
```

#### Step 5: Build & Run

```bash
# Build debug APK
./gradlew assembleDebug

# Install on connected device
./gradlew installDebug
```

1. Tap "Arcade Mode"

3. Choose grid size (3x2 to 6x4)
4. Tap cards to match pairs
5. Complete to see your score

#### Adventure Mode

1. Tap "Adventure Mode"
2. View 16-level map
3. Complete levels to unlock next
4. Earn 1-3 stars based on performance

#### Multiplayer Mode

1. Tap "Multiplayer Mode"
2. Select theme and grid size
3. Players alternate turns (Red vs Blue)

- **Total Games Played:** Lifetime game count

## Troubleshooting

### Installation Issues

**Gradle Sync Failed**

```bash
./gradlew clean
./gradlew --stop
rm -rf ~/.gradle/caches/
```

**Google Services Missing**

1. Download `google-services.json` from Firebase
2. Place in `app/` directory
3. Verify package name: `vcmsa.projects.prog7314`

### Runtime Issues

**App Crashes on Launch**

1. Check Logcat: `adb logcat | grep "AndroidRuntime"`
2. Clear app data
3. Verify Firebase configuration

**Biometric Authentication Not Working**

1. Verify device has biometric hardware
2. Set up fingerprint in device settings
3. Grant biometric permission

**Notifications Not Appearing**

1. Check notification permissions
2. Verify FCM token registration
3. Check quiet hours settings

### Synchronization Issues

**Data Not Syncing**

1. Check internet connection
2. Manual sync: Dashboard → Pull to refresh
3. Verify Firestore rules

**Conflicting Game Data**

1. Force full sync: Settings → Clear Local Cache
2. Logout and login again

### Performance Issues

**App Running Slowly**

1. Clear cache (not data)
2. Close background apps
3. Restart device
